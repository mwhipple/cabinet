***** define_tool

This is mostly cleaned up from previously produced AI input...
mostly to fully conform to the desired format.
Reference sources should be provided which is also something that
should be included in the system prompt.

This does /not/ attempt to be comprehensive. As needed this should be
extended with registries to handle additional types.

#+begin_src python
  import inspect
  def define_tool(func: callable) -> ToolFunctionDefinition:
      """
      Produce a dictionary suitable for llama-cpp-python tool registration.
      """

      description = func.__doc__
      if not description:
          raise TypeError("Function must have a non-empty docstring.")

      sig = inspect.signature(func)
      params = sig.parameters.items()

      return {
          "type": "function",
          "function": {
              "name": func.__name__,
              "description": description,
              "parameters": {
                  "type": "object",
                  "properties": {
                      param_name: {
                          "type": param_type(param),
                      }
                  for param_name, param in params},
                  "required": [param_name for param_name, param in params if is_required_parameter(param)],
              },
              "return_type": str(sig.return_annotation),
          },
      }
#+end_src

*** Class Definition

The ToolBelt object will handle advertising of available tools and
their subsequent use.

Application will produce the message to send back to the model.

#+begin_src python
  class ToolBelt():
      def __init__(self):
          self.tools = {}

      def register(self, func: Callable):
          self.tools[func.__name__] = func

      def definitions(self):
          return [define_tool(tool) for tool in self.tools.values()]

      def apply(self, tool_call: ToolCall) -> ChatCompletionRequestToolMessage:
          if tool_call["type"] != "function":
              raise TypeError("Only functions suppored")
          func = tool_call["function"]
          tool_name = func["name"]
          tool = self.tools.get(tool_name)
          if tool is None:
              raise ValueError(f"Tool {tool_name} not found")
          return apply_tool(func, tool, tool_call["id"])

#+end_src

** ChatDriver

When dealing with lower level interactions, it feels like one of the
main pieces is the component that handles sending messages and dealing
with responses which will be defined here as a ChatDriver.

The focus is that messgage coordination, and providing relevant
extension points to add additional behavior.

The configuration options should be defined as a separate object which
is passed in.

This could certainly benefit from a bit more encapsulation, but to
start its unclear where the best seams are that won't just create a
ball of mud.

Messages will be sent using [[id:C6047B7D-4B82-45B9-886A-FACBF966030B][create_chat_completion]].

on_message will be called on each message received from the model and
serves as the means to trigger most of the logic. This will be passed
the message and a context dictionary (to be more precisely defined).

#+begin_src python
  from llama_cpp import Llama
  from llama_cpp.llama_types import (
      ChatCompletionRequestMessage,
  )

  class ChatDriver:
      def __init__(self,
                   model: Llama,
                   on_response = ForEachHandler()):
          self.model = model
          self.messages: list[ChatCompletionRequestMessage] = []
          self.on_response = on_response

      def append(self, msg: ChatCompletionRequestMessage):
          self.messages.append(msg)

      def submit(self):
          resp = self.model.create_chat_completion(
              messages = self.messages,
              tools = self.tools,
              tool_choice = "auto",
              temperature=0.6,          # Official developer recommendation
              repeat_penalty=1.05,
          )
          choice = single_choice(resp)
          on_response = self.on_response # assign to avoid being a method call
          on_response({"driver": self}, choice)
          self.append(choice['message'])
#+end_src

  import json
  def parameter_from_qwen_match(match):
      name = match.group(1).strip()
      str_value = match.group(2).strip()
      try:
          return (name, json.loads(str_value))
      except json.JSONDecodeError:
          return (name, str_value)

  class QwenToolCallResponseHandler:
      def __init__(self, tool_belt):
          self.tool_belt = tool_belt;

      def __call__(self, ctx, resp):
          calls = parse_qwen_tool_calls(resp['message']['content'])
          if not calls: # Don't submit at the end

          for call in calls:
              driver.append(self.tool_belt.apply(call))
